PrivDoorbell

PrivDoorbell is a open source project, aiming to provide a privacy preserving video doorbell solution.

Technical Overview

Here we present an overview of techniques and security schemes used in this project.

Open source

Check out the source code: GUSecLab/PrivDoorbell - Github

Multiple authentications

PrivDoorbell service adapts several authentication methods.

  • RTMP authentication: Introduced by Nginx-rtmp-module, the system asks the client to provide additional credientials (in the form of GET arguments) in order to access the stream.
  • Onion authentication v2: Provided by Tor, the system requires the client to have certain credientials configured in there Tor/Orbot settings in order to access the stream.

    • Limited access

    In addition to all those authentication schemes, the PrivDoorbell devices are designed to deny all unnecessary access. Limited access prevents attackers from getting control of IoT devices and eliminates a lot of vulnerabilities.

    Port Service Accessibility
    1935 RTMP stream Denied
    8000 Video feed in flv/HTTP Only through Tor, direct access denied
    8080 User portal Only from local network
    8081 Authentication Only through Tor, direct access denied

    Noise feature

    The server sends a message every 5 second regardless of whether there is a visitor or not. By adding these dummy messages, the system makes it safe to utilize third-party pushing service (like Firebase Messaging) without exposing privacy to these providers.
